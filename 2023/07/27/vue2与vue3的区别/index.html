vue2与vue3的区别

1.响应式数据的区别

vue2响应式数据采用**Object.defineProperty()**对数据进行劫持

vue3采用proxy对数据进行代理

2.选项式API和组合式API的区别

vue2采用选项式API，方法和属性都存在Vue上面，直接用就可以了

vue3采用组合式API，方法和属性都封装好了，需要使用时在进行导入，所以vue3更轻更快

3.生命周期函数的区别

vue2生命周期函数：

  1. beforeCreate：在实例初始化之后、数据观测 (data observer) 和事件/生命周期事件配置之前被调用。此时组件的选项和观测的数据都还未初始化。
  2. created：在实例创建完成后被调用。此时组件的选项已经初始化完成，但尚未挂载到 DOM 上，无法访问到 $el。
  3. beforeMount：在组件挂载到 DOM 之前被调用。此时组件的模板已经编译完成，但尚未渲染到页面上。
  4. mounted：在组件挂载到 DOM 后被调用。此时组件已经被渲染到页面上，可以访问到 $el。
  5. beforeUpdate：在组件更新之前被调用，即在重新渲染之前。此时组件的数据已经更新，但尚未应用到 DOM 上。
  6. updated：在组件更新完成后被调用。此时组件的数据已经应用到 DOM 上，可以访问到更新后的 DOM。
  7. beforeDestroy：在组件销毁之前被调用。此时组件仍然完全可用，可以访问到组件实例和其它相关的属性和方法。
  8. destroyed：在组件销毁之后被调用。此时组件已经被销毁，所有的事件监听器和观察者都已经被移除。

vue3生命周期函数：

  1. setup：初始化
  2. onBeforeMount：在组件挂载到 DOM 之前被调用。
  3. onMounted：在组件挂载到 DOM 后被调用。
  4. onBeforeUpdate：在组件更新之前被调用，即在重新渲染之前。
  5. onUpdated：在组件更新完成后被调用。
  6. onBeforeUnmount：在组件卸载之前被调用。
  7. onUnmounted：在组件卸载之后被调用。

4.vue2单根节点，vue3多根节点

vue2的模板里面是需要一个单根节点的

vue3的模板是不需要单根节点的，可以多根节点，vue3的模板在编译之前，会先判断是否是单根节点，如果不是的话，会通过使用一个文档碎片将其包裹Fragment

5.虚拟DOM新增patchFlag字段，方便diff算法

Vue3 相比于 Vue2，虚拟DOM上增加 patchFlag 字段
字段类型情况：1 代表节点为动态文本节点，那在 diff 过程中，只需比对文本对容，无需关注 class、style等。除此之外，发现所有的静态节点（HOISTED 为 -1），都保存为一个变量进行静态提升，可在重新渲染时直接引用，无需重新创建。

6.vue3在diff算法有优化

因为虚拟DOM在patchFlag的优化，diff在对比差异时优化

7.异步组件（没用过）

Vue3 提供 Suspense 组件，允许程序在等待异步组件加载完成前渲染兜底的内容，如 loading ，使用户的体验更平滑。使用它，需在模板中声明，并包括两个命名插槽：default 和 fallback。Suspense 确保加载完异步内容时显示默认插槽，并将 fallback 插槽用作加载状态。

8.Teleport 组件，可以将组件内容移到Vue app外，如Dialog弹窗

Vue3 提供 Teleport 组件可将部分 DOM 移动到 Vue app 之外的位置。比如项目中常见的 Dialog 弹窗。

9.事件缓存机制

Vue3 的cacheHandler可在第一次渲染后缓存我们的事件。相比于 Vue2 无需每次渲染都传递一个新函数。
如果添加一个 click 事件。在第一次渲染的时候，click 事件节点可能会被判断为静态节点（patchFlag为 -1），即不需要每次重新渲染

10.更好的TS支持。

Vue3 由 TypeScript 重写，相对于 Vue2 有更好的 TypeScript 支持。

其实还有很多其他的，一些API名称变化，v-if和v-for的优先级….
